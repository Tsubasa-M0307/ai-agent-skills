# スライド自動作成スキル

Googleスライドのテンプレートをコピーし、Claude in Chrome で編集して仕上げる。

このスキルは Google Slides をブラウザ操作で編集する。最大のコストは「試行錯誤・無駄なスクリーンショット」なので、下記の手順とレシピに従えば、少ない操作で確実に仕上がる。**先に手順を理解してから動くこと**が一番のトークン節約になる。

## 設定情報

- テンプレートファイルID: `1ETj80aigHudtIeLlEf5EePbsu4imKWuCQB8A_wIVzGw`
- 成果物フォルダID: `1hNR_yZjmj2jQS-YUOXswTctMUYtdF7fl`

## Step 1: 必要情報を一度にまとめて確認

未指定なら、以下を**1回の質問でまとめて**聞く（往復を増やさない）。
タイトル / 日付 / 講師名・所属 / 各スライドの内容・構成 / その他変更点（ロゴ・色など）。

## Step 2: テンプレートをコピー

`mcp__b1107aa5-eb71-4769-9a5b-b355b05437cc__copy_file` を使う。
- fileId: テンプレートファイルID
- parentId: 成果物フォルダID
- title: 日本語のみ（英数字混在で文字化けすることがあるので、確定後に必要ならタイトルを直す）

返ってきた `id` と `viewUrl` を記録する。編集URLは `https://docs.google.com/presentation/d/{id}/edit`。

## Step 3: Chrome で編集

`tabs_context_mcp` → `navigate` で編集URLを開く。

### まず全体把握（スクショ1枚）

- フィルムストリップ下部に出る「**合計 N 枚のスライド**」で総枚数を確認する。
- 注意: フィルムストリップのサムネイルは遅延読み込みされる。JS/DOM でスライド数を数えようとしない（不正確で時間の無駄）。`get_page_text` も縦書きで文字がバラけて低品質なので使わない。**構造把握はスクリーンショットが最速**。
- 目的のスライドはサムネイルをクリックして本編キャンバスに表示してから編集する。

### テキスト置換の確実なレシピ（最重要）

各テキストボックスは次の順序で置き換える。これが最も確実で、やり直しが出ない：

```
double_click（対象ボックス内）→ cmd+a → Delete → type（新テキスト）→ Escape Escape
```

理由と注意：
- `double_click` で編集モードに確実に入る。`cmd+a` は**編集モードに入った後**でないとボックス内全選択にならない。
- `Delete` で空にしてから `type` する。**これを省くと旧テキストが残って重複する**（例:「主な機能はじめに」のような事故）。
- `triple_click` は1行選択のつもりでも追記になりやすいので使わない。
- 改行は `type` のテキストに `\n` を入れればそのまま反映される（箇条書き本文を一度に入力できる）。
- タイトルなど1行ボックスでも同じレシピでよい。

### browser_batch でまとめる

1スライド分の編集（サムネイルclick → タイトル編集 → 本文編集 → 確認スクショ）を **`browser_batch` 1回**にまとめる。
- スクリーンショットは**各スライドの最後に1枚だけ**。操作の合間に毎回撮らない。
- バッチ末尾のスクショで結果を確認し、崩れていればその箇所だけ直す。

### リスト（箇条書き）ボックスの扱い

複数行の箇条書きは**1つのテキストボックス**にまとまっていることが多い。その場合 `cmd+a` で全行が選択されるので、`Delete` 後に全行を `\n` 区切りで一度に `type` する。番号付きリストは行を入れれば自動採番される。

### スライドの追加

内容スライドを増やすときは、雛形にしたいスライドのサムネイルを選択して `cmd+d`（複製）。必要枚数だけ複製してから各スライドを上記レシピで編集する。新規白紙より、体裁の整った既存スライドの複製が速く確実。

## Step 4: 完成報告

- 編集URL（`https://docs.google.com/presentation/d/{id}/edit`）
- タイトル・枚数・主な変更点を簡潔に
- 反映しきれなかった箇所があればURLと共に手動編集を依頼

## エラー対応

- Chrome MCP が使えない: 編集URLを共有し手動編集を促す。
- 置換が崩れた: そのボックスを `double_click → cmd+a → Delete` で空にしてから再入力。
- テンプレは編集しない。コピー（成果物フォルダ内）だけを編集する。
